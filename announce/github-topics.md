## Topics to *not* use

These look attractive but don't help, or actively hurt:

- `ai` — too broad, drowns in noise.
- `llm` — same.
- `chatgpt` — wrong audience (most are looking for plugins, not spec).
- `productivity` — too generic; signals "not technical".
- `documentation` — true but unhelpful; hides among 500k other docs repos.
- `methodology` — abstract; nobody searches it.
- `git` — too broad; this isn't a Git tutorial repo.

## Refresh cadence

- **Check every 3 months.** Search behavior shifts; `vibe-coding`
  may become `agentic-coding` next year.
- **Replace, don't append.** Adding a 16th topic forces something
  off the visible list — pick which is least valuable now.
- **Run a real search** for each topic on GitHub Search to verify
  the repo shows up (or doesn't) for that term. If you don't rank
  for your own topics, the topics aren't the problem — the README
  is.

## Bonus: GitHub's "Topic page" SEO

`https://github.com/topics/agents-md` is its own page that ranks
in Google for "agents-md". Owning that page (being the top result
on it) compounds. To be the top result:

- Have the highest star count among repos using the topic
- Have the most recent activity
- Have a README that matches the topic name strongly

Right now this repo will be one of very few in the `agents-md`
topic — likely automatic top spot. Maintain that lead.
